## 二、v3 相比旧版的核心升级

### 结构层面

1. **两层分离**：正文 5 节（PM 5分钟）+ 附录 13 节（研究员复盘答辩）。解决了旧版 18 章平铺、不分优先级的问题。
2. **新增市场背景（②）**：为未来复盘提供时间锚定，旧版完全缺失。
3. **新增分主题分析（④）**：解决多主题视频按统一框架硬套导致的混杂问题。

### 方法层面

1. **Workflow 7 步流程**：`理解主题 → 提炼 → 删除 → 区分事实/观点 → 判断证据 → 判断跟踪价值 → 输出`。本身就是一套研究思维训练。
2. **证据五级评分**：`官方数据/法规/公告 ＞ 财报 ＞ 权威机构 ＞ 多家媒体交叉 ＞ 行业人士 ＞ 视频作者`。给了 LLM 一个可执行的判断树。
3. **5 类逻辑谬误显式检查**：逻辑跳跃 / 偷换概念 / 因果倒置 / 循环论证 / 缺乏证据。旧版只笼统写"逻辑漏洞"。
4. **8 条 Rules 独立成章**：内容过滤 / 时间戳 / 事实优先 / 证据优先级 / 术语解释 / 避免重复 / 独立思考 / 输出原则。用规则约束替代模糊指令。

## 三、v3 仍存在的 5 个问题

### 问题一：缺少信息密度预判机制（最重要）

LLM 拿到 18 节模板就会填充 18 节。如果一个视频从头到尾无独立观点，当前框架仍会产生大量低质量填充。

**建议方案：** 在 Workflow 第 1 步后插入判断门：

> 如果视频符合以下任一情况，直接输出 D（事件噪音），**不展开后续框架**：
> - 全程市场公开信息复述，无独立观点
> - 核心结论建立在不完整数据或单一案例上
> - 主要内容为情绪宣泄、个人盈亏播报
> - 观点与主流共识完全一致，无增量信息

### 问题二：市场共识度分析缺少逐观点表格

v3 的 E 改为叙事格式，但一个视频可能有多个观点且共识度不同。应恢复表格式，针对③中的每个核心结论分别判断。

### 问题三：缺少反方观点维度

D（逻辑链分析）检查了逻辑谬误，但没有强制输出"反对本视频结论的最有力论据"。这是买方研究最关键的思维习惯之一。

**建议：** 在 D 末尾挂载一节反方观点，无需新增独立模块。

### 问题四：合规尾部被删除

旧版的文末免责声明和生成工具标注在 v3 中被删除。需恢复。

### 问题五：User Prompt 模板未同步更新

System Prompt 增加了市场背景和分主题分析，对应的 `USER_PROMPT_TEMPLATE` 需确认是否适配。

## 四、关于"配置化"的评估

ChatGPT 建议了三条路径：分层输出 / 模块化拼装 / 结构化配置层（YAML Schema）。

**逐一评估：**

| 方向 | 解决的问题 | 可行性 | 当前适配度 | 推荐 |
|------|-----------|--------|-----------|------|
| A. 分层输出（Quick / Standard / Deep） | Prompt "太重" | 极高，改 User Prompt 即可 | ★★★★★ | ✅ 立即做 |
| B. 模块拼装（按视频类型选模块） | 不同内容适用不同框架 | 中，需先定义类型映射 | ★★☆ | ⚠️ 等真实需求出现 |
| C. YAML 配置化 | 配置与代码分离 | 高，但需写渲染层 | ★☆☆ | ❌ 当前过度工程化 |

**A 是最务实的下一步。** 在 User Prompt 中加一个深度参数，不重构架构，10 分钟完成。

## 五、ChatGPT 三段式架构评估

ChatGPT 建议了三个阶段演进：

### Stage 1：Information Density Gate → ✅ 完全同意

与问题一一致。零额外 API 调用，零架构变动。应该做。

### Stage 2：gate.md / research.md / formatter.md Pipeline → ⚠️ 方向对，分法有问题

ChatGPT 隐含假设三个阶段需要三次 LLM 调用：

| 指标 | 当前（单体） | 3 次调用 Pipeline |
|------|------------|-------------------|
| API 调用次数 | 1 | 3 |
| 延迟 | 30-60s | 90-180s |
| 失败概率 | 单点 | 任意一环失败则整体失败 |

**真正有 ROI 的拆分是按模型能力需求，而非按功能步骤：**

```
Phase 1: deepseek-reasoner（推理模型）
  → Gate 判断
  → 提炼核心结论
  → 区分事实/观点
  → 逻辑链分析
  → 输出结构化 JSON

Phase 2: deepseek-chat（便宜 10 倍的模型）
  → 接收 Phase 1 的结构化结果
  → 按模板渲染 Markdown
```

这才是真正的降本增效——推理模型做思考，便宜模型做填充。

### Stage 3：planner.md → AI 动态决定模块 → ❌ 危险

三个核心反对理由：

1. **知识库的核心价值是结构一致性。** 当三个月后回看多篇笔记时，"E. 市场共识度"一定在同一个位置才能横向比较。AI 动态决定模块意味着每篇结构不同，可比性被破坏。
2. **LLM 不擅长结构决策。** LLM 判断"这个视频需要输出 F（投资影响分析）吗"的准确率不会超过 70%。而固定框架的准确率是 100%，代价只是偶尔输出"本模块无相关信息"。宁可有空模块，不可丢信息。
3. **Agent ≠ 去掉约束。** Agent 的正确范式是：固定目标 + 固定约束 + 自主执行路径。v3 的 Workflow 7 步已经做到了执行路径自主。不需要再让 AI 决定模块取舍。

## 七、关键设计原则（存档）

1. **不要为假设的未来需求做架构。** 当前的唯一使用场景是 B 站财经视频总结，按这个场景优化到极致。
2. **结构一致性 > 灵活性。** 知识库的长期价值来自可比较、可检索的固定框架。
3. **宁可有空模块，不可丢信息。** 固定框架的遗漏率是 0%，动态模块的遗漏率不可控。
4. **Agent 的本质是自主执行路径，不是自主决定目标或约束。**
5. **Pipeline 拆分应按模型能力需求（推理 vs 渲染），而非按功能步骤（gate vs research vs formatter）。**
